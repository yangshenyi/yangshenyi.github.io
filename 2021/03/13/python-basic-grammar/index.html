python是一种面向对象的解释型语言。

Beautiful is better than ugly.

对象

python中一切皆为对象。

我们通过描述属性（特征）和行为来描述一个对象的。比如家里的小狗，它的颜色，大小，年龄， 体重等是它的属性或特征。它会汪汪叫，会摇尾巴等是它的行为。 我们在描述一个真实对象（物体）时包括两个方面： 它可以做什么（行为） 它是什么样的（属性或特征）。

在python中，一个对象的特征也称为属性（attribute）。它所具有的行为也称为方法（method）

类和对象

​ 对象表示一个实体，每个对象都有自己的标识、属性和行为。在这里，属性特指数据域；行为，也叫动作，是由方法确定的，这样，调用对象的方法，就是让他发生一个动作。

​ 类是定义属性和行为的模板。python中，万物皆为对象，所谓类，有点像我们在C里面学的struct，但是那时候里面只有属性而没有方法，换言之，类就是封装了变量和方法的一种数据类型。不同的对象可以有不同的数据域，占据了内存，是类的一个实例。

​ 面向对象程序设计就是要评估待解决问题，恰当的设计类，然后创建类的实例。这些普遍性的类可以封装成库，供更多的人使用。

    • 创建： 类名（参数） 返回一个对象的引用
    • 使用：
      • 对象成员包含数据和方法。一个方法可以被一个对象调用来完成它的一个动作（也许是修改属性）
      • 对象名字.成员
      • 类中的每个属性
        • 都得有一个初始值（创建的时候能关联到一块内存，不然这个属性就不存在。或者说，我们不必像C++那样specifically声明属性，而是在使用时候self.?声明了），或者是参数，或者是在初始化函数里面关联。
        • 至于修改属性，可以通过对象引用，也可以通过方法间接修改
        • pass关键字可以表示空语句，用在方法定义中

特性

  • 封装

    将属性、方法集合在一个整体里面，通过访问控制，同时可以设置权限，将实现的细节对用户隐藏，只提供接口。于是，类也叫做抽象数据类型或者复合数据类型。

    • 成员

      • 数据

        实例成员。在init里面定义，每一个对象都有其副本。类内使用self访问，类外使用对象

        类变量（静态变量）。属于这类，在类定义中，方法外定义。所有对象（实例）共享它。对象或者类名都可以访问它。

      • 方法

        实例方法。

        类方法。不能对特定的实例操作，也自然不能访问实例变量。

      ​ 静态方法

    • 权限

      python类的成员没有像C++那样有明确的访问控制，但是有一些约定。

      一个下划线前缀_。受保护成员，不能from module import *导入

      两个下划线和两个下划线前后缀。系统定义的特殊成员（init）

      只是两个下划线为前缀。私有成员，只能类内访问。

​ 在python里面，想要访问私有成员，必须用装饰器修饰了的方法。

​ @property默认只提供 只读权限

继承 ＆ 多态

  • 继承是代码重用的一种重要手段/创建新的类的机制。子类可以继承父类的属性和方法，也可以对此修改，还可以添加。

  • 使用

    在子类的定义中，名字后的括号里面填父类的名字

    super()是一个特殊的函数，可以将父类和子类关联起来，使得子类的init可以调用父类的构造函数，实现父类数据成员的初始化。

  • 多态

    动态绑定。程序运行时，由对象的类型确定变量的类型。比如 ：类重载操作符

异常处理&&测试

异常

python运行发生错误时，它都会创建一个异常对象。如果你编写了处理该异常的代码，程序将继续运行；如果未对异常进行处理，程序将停止并显示traceback，其中包含有关异常的报告

测试

​ 可以设置死循环（只有当按某些键值才会退出），”搭建测试环境“，手动尝试不同的输入。

​ 可以使用unittest模块自动化测试。

    • 导入模块
    • 创建自定义类（继承unittest.TestCase）
    • 定义测试函数（使用assertEqual方法判断结果是否正确）
